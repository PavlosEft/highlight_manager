Έλεγχος των Highlights (Offsets & Undo/Redo)
Το Python project προσφέρει πολύ πιο granular έλεγχο στο χρήστη όσον αφορά την επεξεργασία των φάσεων.

Custom (Ατομικά) Offsets ανά Highlight: Στο Python, κάθε highlight έχει τα δικά του βελάκια ◀ 2s ▶ και ◀ 3s ▶ για να ρυθμίζει ο χρήστης το start/end offset μεμονωμένα (και κουμπί ↺ για επαναφορά). Στο Flutter υπάρχουν μόνο τα γενικά (global) sliders.

Άμεσο Preview στα Offsets: Όταν στο Python άλλαζες το ατομικό offset, το βίντεο έκανε αυτόματα seek στο νέο σημείο για να κάνεις άμεσο preview (μέθοδος adjust()).

Σύστημα Undo/Redo: Το Python διαθέτει πλήρες σύστημα αναιρέσεων (history_stack, redo_stack, action_stack) με αντίστοιχα κουμπιά για να παίρνεις πίσω ενέργειες (π.χ. όταν μεταφέρεις μια φάση κατά λάθος στα Highlights). Το Flutter δεν έχει undo/redo.

Έξυπνο Ιστορικό Πλοήγησης (N / G): Το Python θυμάται με ακρίβεια τη σειρά των φάσεων που πάτησες (forward_stack, history_stack), επιτρέποντας σωστή πλοήγηση μπρος-πίσω με τα πλήκτρα N/G. Το Flutter απλά αυξομειώνει το index (activePhaseIndex + direction).

Select All / Deselect All: Στο Python υπάρχει checkbox δίπλα στον τίτλο των Highlights για μαζική επιλογή/αποεπιλογή φάσεων, κάτι που λείπει από το Flutter.

Reset Seen Status: Το Python διαθέτει κουμπί "Καθαρισμός" (Reset Seen) για να σβήσει μαζικά το seen status από όλες τις φάσεις. Στο Flutter μπορείς μόνο να τις κρύψεις (Hide Seen).

3. Διαχείριση Αρχείων & Projects
Η εμπειρία προσθήκης και φόρτωσης αρχείων είναι πιο πλούσια στο παλιό σύστημα.

Smart Relinking (Αναζήτηση Αγνοουμένων Αρχείων): Αν στο Python διαγραφεί ή μετακινηθεί ένα αρχείο βίντεο, το πρόγραμμα βγάζει prompt για να το εντοπίσεις χειροκίνητα. Αν το βρεις, ψάχνει αυτόματα και τα υπόλοιπα χαμένα αρχεία στον ίδιο νέο φάκελο (smart relinking). Το Flutter απλά θα αποτύχει να το φορτώσει.

Πληροφορίες στο New Project Dialog: Το Python διαβάζει μέσω ffprobe (ή os.path) και εμφανίζει σε στήλες τη Διάρκεια, το Μέγεθος (MB/GB) και την Ημερομηνία Τροποποίησης των βίντεο πριν τα προσθέσεις. Το Flutter δείχνει μόνο το όνομα του αρχείου.

Αποθήκευση τελευταίου φακέλου Export: Το Python θυμάται τον φάκελο που έκανες export την τελευταία φορά (μέσω QSettings). Στο Flutter πρέπει να τον επιλέγεις από την αρχή κάθε φορά.

4. Εμπειρία Χρήστη (UI/UX) & Player
Πολλές ευκολίες διασύνδεσης (UI) δεν έχουν μεταφερθεί ακόμη.

Dual List View vs Unified List: Το Python διαχωρίζει ξεκάθαρα τις φάσεις σε "Detected (Pool)" (Αριστερά) και "Highlights" (Δεξιά) με κουμπιά ▲/▼ για μεταφορά. Το Flutter χρησιμοποιεί μία ενιαία λίστα με "αστεράκια" (isHighlight) και κουμπιά φιλτραρίσματος (Highlights Only / Hide Seen). Πρόκειται για διαφορετική σχεδιαστική προσέγγιση, αλλά χάνεται η αίσθηση του "μεταφέρω υλικό στο τελικό καλάθι".

Fullscreen & Overlay Controls: Το Python διαθέτει κουμπί Fullscreen (⛶), το οποίο κρύβει όλο το UI και εμφανίζει ημιδιάφανα overlay controls (play/pause, slider) στο κάτω μέρος της οθόνης.

Έλεγχος Έντασης Ήχου: Το Python έχει κουμπί Mute (🔊/🔇) και Slider έντασης (0-100). Στο Flutter υπάρχει μόνο το Play/Pause και η μπάρα χρόνου.

Οπτικό Υπόμνημα (Legend): Στο κάτω μέρος της Python οθόνης υπάρχει ένα μόνιμο υπόμνημα που θυμίζει τα Shortcuts (Space, H, N, G), το οποίο λείπει από το Flutter (αν και τα shortcuts δουλεύουν κανονικά κρυφά στο background).
